# 链表
基础的动态数据结构体

综合时间复杂度为O(n)，适用于**针对表头**进行一系列操作的场景，如增、删、查。这三种操作时间复杂度为O(1)。**但不推荐修改**

因为链表虽然有索引(index)，查找效率低下，时间复杂度为O(n)，不像数组那样根据索引就能直接找到元素

相比较创建数组，链表整体是动态的，所以不会浪费大量的内存空间。


## 基本概念

## 操作
### 增
综合时间复杂度：O(n)

- 表头操作：O(1)
- 表尾操作：O(n)


### 删
时间复杂度：O(n)

- 表头操作：O(1)
- 表尾操作：O(n)

### 改
时间复杂度：O(n)

### 查 
时间复杂度：O(n)
- 表头操作：O(1)
- 表尾操作：O(n)
